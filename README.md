# IFC-account

部活動の会計を管理するWebアプリです。Googleアカウントでログインし、Googleスプレッドシートをデータベースとして収支・部員・部費・予算を一元管理します。

## 主な機能

- **ホーム（ダッシュボード）**：月間収支のサマリー表示
- **収支入力**：収入・支出の登録
- **帳簿**：取引履歴の閲覧・編集・削除
- **部員管理**：部員情報の一括登録、期間別属性（学年・OB/OGなど）の管理
- **部費管理**：部員ごとの部費回収状況の管理
- **予算管理**：コート代・その他カテゴリの予算設定、月間/年間ビューの切り替え
- **レポート**：カテゴリ別・月別の収支傾向表示
- **カテゴリ管理**：収支カテゴリの編集
- **バックアップ**：スプレッドシート全体をワンクリックで複製
- 年度（会計年度）切り替えに対応

## 技術構成

- フロントエンド：HTML / CSS / JavaScript（`index.html`, `static/js/`（機能別に分割されたJavaScript）, `static/style.css`）
- 認証：Google Identity Services（OAuth）
- データストア：Google Sheets API v4（スプレッドシートをDBとして利用）

## セットアップ手順

このアプリを別のGoogleアカウント・別のスプレッドシートで使う場合は、以下の準備が必要です。

1. **OAuthクライアントIDの発行**
   - [Google Cloud Console](https://console.cloud.google.com/) でプロジェクトを作成（または既存プロジェクトを選択）
   - 「APIとサービス」→「ライブラリ」で **Google Sheets API** と **Google Drive API**（バックアップ機能で使用）を有効化する
   - 「APIとサービス」→「認証情報」から OAuth 2.0 クライアントID（ウェブアプリケーション）を発行
   - 「承認済みのJavaScript生成元」にアプリを配信するURL（例：`http://localhost:8080`や本番URL）を追加
   - 発行されたクライアントIDを `static/js/config.js` の `CLIENT_ID` に設定する
   - ※バックアップ機能追加に伴いOAuthスコープに`drive.file`を追加したため、既存ユーザーは次回ログイン時に権限の再同意が必要になります
2. **スプレッドシートの用意**
   - 新しいGoogleスプレッドシートを作成し、そのスプレッドシートIDを `static/js/config.js` の `SHEET_ID` に設定する（初回起動時に必要なシート・ヘッダーはアプリが自動で作成します）
3. **利用者アカウントへの権限付与**
   - 会計担当者のGoogleアカウントに、上記スプレッドシートの**編集権限**を付与する

### ローカル起動

```bash
python3 -m http.server 8080
```

## 運用上の注意

- **スプレッドシートを直接開いて行の挿入・削除・並べ替えをしないこと**
  アプリは「行番号（スプレッドシートの行順）＝データの識別子」という前提で動作しています。直接編集で行の順序がずれると、アプリ側のデータと不整合が起き、**データが破損します**。値の修正が必要な場合も、行そのものを動かさないよう注意してください。
- **複数人が同時にアプリを開いて編集しないこと**
  保存は行単位のAPI呼び出しに分けられており、シート全体を洗い替える方式に比べれば安全になっていますが、同じデータを同時に編集した場合の排他制御（ロック）はありません。複数人が同時に操作すると、後から保存した側の変更で上書きされる可能性があります。会計担当は基本的に1人ずつ操作してください。
